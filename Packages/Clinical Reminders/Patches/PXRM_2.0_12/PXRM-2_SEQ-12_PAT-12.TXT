 General Overview:
 =================
 The build for this patch is distributed as part of a multi-package build that
 contains PXRM*2.0*12, GMTS*2.7*89, OR*3.0*295, and TIU*1.0*249.
 
 This patch contains several major enhancements, many minor
 enhancements, and also bug fixes. This overview summarizes
 the contents of this patch. Complete details can be found in the
 Installation Guide and the Clinical Reminders Manager's Manual that
 are distributed with this patch.
 
 Reminder Exchange has been enhanced to allow selection of individual
 reminder components, in any combination, to be packed into a single
 Exchange entry.
 
 Multiple national reminder components are updated. The Exchange file
 entries installed by this patch are:
 
  PATCH 12 ITEMS
  UPDATE VA-DIABETES
  VA-EMBEDDED FRAGMENTS RISK EVALUATION
  VA-MHV HIGH RISK TERMS
  VA-MHV RETINOPATHY TERMS
  RT VA-ALCOHOL NONE PAST 1YR
  TX HIGH RISK FLU AND PNEUMONIA
  VA-ALCOHOL AUDIT-C POSITIVE F/U EVAL
  VA-DEPRESSION SCREENING
  VA-OEF/OIF MONITOR
  VA-MHV MAMMOGRAM SCREENING
  VA-TBI/POLY IDT EVALUATIONS ELEMENT UPDATE
  VA-MHV BMI COLORECTAL UPDATES PATCH 12
  VA-BL DEPRESSION SCREEN
  PXRM VISIT DATE MONTH REQ YEAR BLANK
  VA-OEF/OIF MONITOR REPORTING
 
 See the Installation Guide, PXRM_2_12_IG.PDF, for complete
 instructions.
 
 In support of the Pharmacy Reengineering Project, direct read of
 Pharmacy files has been replaced by API calls.
 
 Initial changes to support standardization of reminder findings are
 included in this patch. The final changes will be made by patch
 PXRM*2*14 which will be released in conjunction with the
 standardization of Immunizations and Skin Tests.
 
 A new finding usage report is included. The report can be used to
 determine where selected finding(s) are used in definitions,
 dialogs, and terms.
 
 The national computed finding VA-BMI is changed from a single
 occurrence to a multi-occurrence computed finding. This does not
 require any changes to existing terms and definitions that use this
 computed finding. It allows for more flexible date range searching,
 which makes it easier to use in reminder patient lists. A new
 computed finding, VA-BSA, which also uses height and weight data,
 provides a way to determine body surface area. The VA-PROGRESS NOTE
 computed finding was changed to allow either the exact name of the
 note title or the internal entry number of the note title.
 
 Clinical Maintenance output for Radiology findings now includes
 report status.
 
 Finding date was added as a "CSUB" data for all reminder finding
 types. "DIASTOLIC" and "SYSTOLIC" were added as "CSUB" data for
 blood pressure findings.
 
 RANK_DATE was added as a new function that can be used in a Custom
 Date Due.
 
 A computed finding inquiry was added and the formatting of the
 inquiry outputs for the various reminder components was made more
 consistent.
 
 Automatic taxonomy expansion checking was added as part of taxonomy
 inquiry. Options to verify all taxonomy expansions and to expand all
 taxonomies were added to the taxonomy management menu.
 
 For a number of years there have been reports that the output from
 Reminder Due Reports occasionally didn't show up. This problem was
 not reproducible and a number of things were done to try to find the
 cause. It appears that a possible cause is some sites have created
 aggressive cleanup routines that delete tasks that are not
 scheduled; this discovery led to a restructuring of reminder due
 reports. Previously, two tasks were created: the first assembled the
 data and the second produced the output. In order to not tie up the
 output device while the data was being assembled, the print job was
 not scheduled to run until the assembly job finished. In the new
 structure, the assembly and print job are combined, but the output
 device is not opened until it is ready to output the data.
