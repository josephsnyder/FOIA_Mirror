Subject: UPDATE DATA EXTRACT FIELDS FOR DSS

Category:

Description:
============

 **NOTE** This patch includes changes to the LAB package related to Fiscal
 Year (FY) 11 updates to the Decision Support System (DSS) package that are
 included in patch ECX*3*127.  This patch should be installed immediately
 before installing patch ECX*3*127.
  
 This patch adds new fields to the WKLD LOG FILE (#64.03) file in order
 for the LABORATORY EXTRACT for FY11 to extract the following data.
  
 Lab Billable Procedure (#18) 
 Field length = 1
 Source: WKLD CODE file (#64), BILLABLE PROCEDURE field (#4)
 Values: DSS expects a 'Y' for Yes, 'N' for No.
            
 Data definition: If a "Y" is entered it will indicate that the test is
 considered to be a billable LMIP procedure (lab billable means Lab staff
 workload).  
  
 DSS Feeder Key (#19)
 Field length = 1
 Source:  WKLD CODE file (#64), DSS Feeder Key field (#14)
 Values:  DSS expects a '1' for Yes, or null if the source field is null.
 Data definition:  If a "Y" is entered it indicates the WKLD Code should be
 used for DSS Laboratory data collection.
  
 Data Name (#20)
 Field length = 3-40 characters
 Source:  LABORATORY TEST file (#60), NAME field (#.01)
 Values:  free text
 Data definition: This field is the Lab Test name.
  
  
 Data Location (#21)
 Field length = 4-12 characters
 Source:  LABORATORY TEST (#60) file, LOCATION (DATA NAME) field (#5)
 Values:  free text
 Data definition: The Data Location field is only populated if a test is a
 non-panel test  and is part of the CHEM, HEM, TOX, SER, RIA, etc. area.
