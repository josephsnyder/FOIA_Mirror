Description:
============

 This patch supports the Class III to Class I conversion of the new GUI 
 application, "Mobile Electronic Documentation" (MED).
 
 This patch also addresses the issue when packing a reminder dialog and the user 
 was unable to finish the process because they would receive the following
 erroneous message:
 
 "The element <Dialog Element Name> contains a reference to a TIU Object 
 <Object Name> in the Dialog/Progress Note Text Field. This TIU Object does not
 exist on the system."
 
 This message appeared because the object was defined in the dialog by the 
 object's abbreviation name and not the object name.  The code will now check
 both the name and the abbreviation fields to determine if the TIU Object is
 valid.  The packing of TIU Objects will still follow the follow the rules
 defined in PXRM*2.0*12.
 
 Build Components:
 ================
 The post-install routine TIUPS244 will automate several tasks to prepare the 
 environment for MED.
 
   - Creates a new template folder called Mobile Electronic Documentation and a
     template called Import M.E.D. Notes for importing the MED notes
     in the "Shared Templates" folder.  If your site renamed the Shared 
     Templates folder, this step will have to be done manually.
 
     NOTE:
 
   - When TIU*1*244  is installed, it will look for the "Shared Templates" 
     shared root container TYPE  in the TIU Template File #8927.  You MUST 
     have only one entry "Shared Templates" for the install to work properly.
     If there is more than one entry "Shared Templates", you MUST identify 
     the national root container  in the TIU Template File #8927 and rename 
     the other "Shared Templates" folders temporarily for the install. 
 
   - A new TIU template folder is created under Shared Templates 
     "Mobile Electronic Documentation."
 
   - A TIU template titled "MED Notes" used for importing the MED notes in 
     the "Mobile Electronic Documentation" templates folder is added.
 
   - The included template is added as a COM object.
 
 Options:
 
   - MANAGE MOBILE ELECTRONIC DOCUMENTATION [TIU MED MANAGEMENT]
     Provides the secondary menu option for selecting the desired Health
     Summary Report.  Assign this option to the required individual's 
     secondary menu option.
 
   - HEALTH SUMMARY PARAMETER DELETE [TIU MED DEL PARM]
     Provides the secondary menu option for deleting the health summary for
     the desired parameter.  Assign this option to the required individual's
     secondary menu option.
 
   - Mobile Electronic Documentation v1.0 [TIU MED GUI RPC]
     Creates the desired context for the GUI application on the server.
 
 Security Keys:
 
   - TIU MED MANUAL OVERRIDE:  Allows user to manually add patients instead of
                               requiring lookup*
   - TIU MED MANUAL PATIENT:  Allows user to manually add patients instead
                              of requiring lookup*
   - TIU MED MGT:  Allows user to manage MED options for Health Summaries
 
 * The MANUAL PATIENT key allows entry where the first letter of the last name
   and the first letter of the first name, DOB & full SSN must match exactly.
   The MANUAL OVERRIDE key allows entry where the full SSN must match exactly.
   (Please see the documentation for further explanation.)
 
 Remote Procedures:

 Installation Instructions:
 =========================
 This patch may be installed with users on the system.
 Installation of this patch should take less than 2 minutes.
  
 1.  Use the INSTALL/CHECK MESSAGE option on the Packman Menu.  
   
 2.  From the Kernel Installation and Distribution System (KIDS) Menu, 
     select the Installation menu.  
   
 3.  From the Installation menu, you may elect to use the following 
     options (when prompted for INSTALL NAME, enter TIU*1.0*244): 
  
     a.  Backup a Transport Global 
     b.  Compare Transport Global to Current System 
     c.  Verify Checksums in Transport Global
  
 4.  If prompted 'Want KIDS to Rebuild Menu Trees Upon Completion of
     Install? YES//', respond NO.
  
 5.  If prompted 'Want KIDS to INHIBIT LOGONs during the install? YES//',
     respond NO.
  
 6.  If prompted 'Want to DISABLE Scheduled Options, Menu Options, and 
     Protocols? YES//', respond NO.
  
 7.  If prompted 'Delay Install (Minutes):  (0-60): 0//', respond '0'.
 
 Post Install Instructions:
 =========================
 Please refer to the Mobile Electronic Documentation (MED) for full details.
 
 Before using the GUI application:
 
      - The desired Health Summary Report must be
         entered using the MANAGE MOBILE ELECTRONIC DOCUMENTATION [TIU MED 
         MANAGEMENT] option.  The report may be set at the USER, SERVICE,
         DIVISION and/or SYSTEM levels.
 
      - Users must be given the appropriate security keys.
 
      - The import template should be verified in CPRS.  If the template is
        not present, the pre-install was not able to uniquely identify the
        "Shared Templates" folder.
