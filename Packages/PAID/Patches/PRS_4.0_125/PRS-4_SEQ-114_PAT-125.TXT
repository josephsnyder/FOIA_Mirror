Subject: NIGHT DIFF CALCULATION; APPROVE SIGNED ESR'S 

Category: 
  - Routine

Description:
============

 This patch addresses 2 issues:
 1. Leave Without Pay (LWOP) is not being counted in the Night Differential
    (ND) calculation
 2. Hard error occurs in Approve Signed ESRs [PRSP ESR SAP] option when
    the Time & Leave (T & L) unit begins with an alpha character
  
  
 Associated NSR(s):
 ==================
 N/A
  
  
 Associated Remedy ticket(s):
 ============================
 1. HD0000000055389 - Leave Without Pay (LWOP) is not being counted in
                      Night Differential (ND) calculation for General
                      Service (GS) employees
    Duplicate:        HD0000000055525 -- Night Differential Problem
  
 2. HD0000000322302 - ONEPTP+11^PRSPSAPU error in Approve Signed ESRs [PRSP
                      ESR SAP] option
  
  
 Participating Test Sites:
 =========================
 Grand Junction VAMC
 Heartland East HCS
 Heartland West HCS

 Remedy Overview:
 ================
 1. HD0000000055389 - LWOP is not being counted in ND calculation for GS 
                      employees
  
 Problem:
 --------
 Night Differential hours are incorrect because Leave Without Pay (LWOP)
 is not being counted in the calculation.
  
 Resolution:
 -----------
 Routine PRS8AC was modified to include LWOP as a valid leave type when
 calculating night differential.
  
  
 2. HD0000000322302 - ONEPTP+11^PRSPSAPU error in Approve Signed ESRs 
                      [PRSP ESR SAP] option
  
 Problem:
 --------
 When using Approve Signed ESRs [PRSP ESR SAP] option a user received a
 hard error when entering a Time & Leave (T&L) unit that began with an
 alpha character.
  
 Resolution:
 -----------
 Routine PRSPSAPU was modified to correct the code that did not return a
 value during a function call which was causing the hard error.
