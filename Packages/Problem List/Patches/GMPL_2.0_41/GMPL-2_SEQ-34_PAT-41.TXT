Subject: CORRECT WINDOWS REPORT FOOTER

Category: 
  - Routine

Description:
============

 This patch corrects the printing of inaccurate CBOC division information
 in the footer section of the Patient Problem List report which is printed
 from the Problems Tab in the CPRS GUI.
  
 Associated NSR(s):
 ==================
 N/A
  
 Associated Remedy Tickets:
 ==========================
 HD0000000394241 - Printed Problem List displays inaccurate division info
   
 Participating Test Sites:
 =========================
 Veterans Health Care System of the Ozarks - Fayetteville
 VA North Florida/South Georgia VA HCS
 VA Palo Alto HCS
  
 Remedy Overview:
 ================
 HD0000000394241 - Printed Problem List displays inaccurate division info
  
 Problem:
 --------
 From the CPRS GUI, when a user that is logged in with a CBOC division 
 prints the PROBLEM LIST report, the division information in the footer 
 is either incorrect or no division information is displayed. This report
 is printed using the Print option from the File menu in the Problems Tab
 in the CPRS GUI.
   
 Resolution:
 -----------
 Modification of routine GMPLPRNT to use the REGISTRATION package API 
 $$SITE^VASITE to return the user's VAMC division at the date/time the
 problem list report is run in the Problems Tab of CPRS GUI in the footer
 of the report.
