  FUNCTIONALITY:
  ============== 
  This patch addresses errors in the MHA3 HL7 messaging feature and allows 
  better follow-up and reporting of HL7 issues. Re-sending Application 
  Rejects after an appropriate delay is enabled. A change to the Internet
  Protocol (IP) address was required.
  
  These changes will be seen by the few super users who have access to the
  MHS Manager [YSMANAGER] menu. Several options to better manage MHA3 HL7
  messaging are available as documented below.
  
  
  Select MHA3 Utilities Option: ???
  
  'Delete Patient Data'     Option name: YSMKIL     Synonym: 3
       This option allows staff to delete patient test data from all or
       selected tests.  The tests may be completed or incomplete.
       This option is a restricted management function.
  
  'Detailed Definition'     Option name: YTQ DETAILED DEFINITION     
  Synonym: 2
       Displays all the information available for a specified MHA3
       instrument. This includes Intros, Questions, Choicetype, Choices,
       Skips, Skip rules, and Reports.
  
  'Exempt Test'     Option name: YTQ EXEMPT TEST     Synonym: 5
       Set restrictions on Administering and Reporting of individual tests
       base on the security keys held by the ORDERER of the test.
  
       The number of days that a test can be re-started if incomplete can 
       also be changed.
  
  'MHA3 HL7 Utilities'     Option name: YTQ HL7 UTILITIES     Synonym: 8
       Menu of options for examining HL7 messaging in MHA3.
  
  'Print Test Form'     Option name: YTQ FORM PRINT     Synonym: 1
       Displays a representation of the test as it might look on a paper 
       form. This is NOT the way the published test form looks but it will
       print Introductions, Questions and Choices.
  
  'Stop/Re-Start Progress Notes for an Instrument'     Option name: YTQ 
  PNOTE FLAG     Synonym: 4
       Allows site to turn off MHA3 generated TIU notes for a specified 
       instrument.
  NOTE: This option will be OUT OF ORDER for now. Its functionality will
        be made available in a future patch.
  
  'Test Usage'     Option name: YTQ USAGE     Synonym: 6
       Returns number of completed administrations by test for a specified
       period.
  
  'XML Output'     Option name: YTQ XML OUTPUT     Synonym: 7
       Outputs an XML document of MHA3 Administration data filtered by:
       
                 D         Date Only
                 I         Instrument
                 L         Location
                 P         Patient
                 O         Ordered By
       
       All reports have a date range. You may include results and 
       individual question data.
       
       This report is in XML format to aid use by other reporting tools. 
       Output contains Patient Identified Data and its use is restricted to
       those who meet all requirements for data security and research and
       clinical authorization.
  
  Select MHA3 HL7 Utilities Option: ???
  
  'HL7 Check'     Option name: YTQ HL7 CHECK     Synonym: 1
       Check the status of the MHA3 HL7 link and the status of an 
       individual message.
  
  'Re-Send individual HL7 Message'     Option name: YTQ HL7 SEND1     
  Synonym: 2
       Re-Send and individual HL7 message. The message is selected by 
       patient, then test and then by date.

  INSTALLATION INSTRUCTIONS:
  ==========================
  This patch may be loaded with users on the system.
  Installation will take less than 2 minutes.
  
  1) Use the HL menu option Start/Stop Links [HL START] to shutdown
     the "YS MHAT" Logical Link Protocol (LLP) if it is running.
   
  2) Use the INSTALL/CHECK MESSAGE option on the PackMan menu.
     (Note:  TEXT PRINT/DISPLAY option in the PackMan menu will
             display the patch text only.)
  
  3) From the Kernel Installation and Distribution System Menu, select
     the Installation menu.
  
  4) From this menu, you may elect to use the following options:
     (when prompted for INSTALL NAME:  enter YS*5.01*97)
       a.  Backup a Transport Global - this option will create a backup
           message of any routines exported with the patch. It will NOT
           backup any other changes such as DD's or templates.
       b.  Compare Transport Global to Current System  - this option will
           allow you to view all changes that will be made when the patch
           is installed. It compares all components of the patch.
           (routines, DDs, templates, etc.)
       c.  Verify Checksums in Transport Global - this option will
           allow you to ensure the integrity of the routines that
           are in the transport global.
       d.  Print Transport Global - this option will allow you to view the
           components of the KIDS build.
  
  5) Use the Install Package(s) option and enter: YS*5.01*97
  
  6) When prompted "Enter the Coordinator for Mail Group 'YS MHA-MHNDB':" 
     either take the default if there is one or enter a name.
  
  7) When prompted "Want KIDS to Rebuild Menu Trees Upon Completion of 
     Install? NO//" respond according to your facility's policy concerning
     the rebuilding of menu trees.
  
  8) When prompted "Want KIDS to INHIBIT LOGONs during the install? YES//",
     respond, "NO".
  
  9) When prompted "Want to DISABLE Scheduled Options, Menu Options, and
     Protocols?  YES//", respond, "NO".
  
  POST-INSTALLATION INSTRUCTIONS:
  ===============================
  Use the Start/Stop Links [HL START] option to restart the "YS MHAT" LLP.
